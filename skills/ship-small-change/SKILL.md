# Skill: ship-small-change

## Purpose

Deliver a scoped change in the smallest safe unit.

## Inputs

- Approved task spec
- Target code area
- Validation checklist

## Steps

1. Confirm scope and acceptance criteria.
2. Implement minimal cohesive code change.
3. Run required checks.
4. Summarize changes and residual risk.

## Output

A small, validated change with a clear summary.

## Quality Checks

- Change set is tightly scoped.
- Validation evidence is included.
- Rollback path is known.
